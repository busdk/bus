# Roadmap 0.0.46 — HTTP: `GET /v1/micropayments/report` (Report / Aggregate)

## Goal
Add a hosted HTTP endpoint that exposes micropayments reporting over normalized transactions by delegating to the existing core reporting operation.

## Progress
planned

## Non-goals

- Define new aggregation semantics beyond the micropayments spec.

## User-visible outcomes
- `GET /v1/micropayments/report` returns deterministic aggregates.

## Related previous versions
- `0.0.32` (core micropayments report operation exists)
- `0.0.44` (HTTP server skeleton exists)

## Canonical definitions

## Implementation goals
- Define query params + response schema in OpenAPI.
- Implement handler that maps query params to the core report operation.

## Acceptance criteria
- Output matches core report operation exactly for the same inputs.
- Endpoint is tenant-scoped and deterministic.
