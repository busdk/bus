# Roadmap 0.0.12 — CLI Command: `bus schema init`

## Goal
Add the `bus schema init` CLI command as a thin wrapper over the core schema init operation.

## Progress
planned

## Non-goals
- Any new schema logic in core.

## User-visible outcomes
- `bus schema init ...` creates a schema file and registers it in the manifest.

## Related previous versions
- `0.0.11` (core schema init exists)
- `0.0.5` (CLI scaffolding exists)

## New or changed interfaces (exact names + boundaries)
None.

## Module responsibilities (what belongs in a module vs core)
- **Core**: owns schema init operation.
- **CLI**: parses args and calls core.

## Data model changes
None (delegates to core).

## API/CLI changes
- `bus schema init <schemaName> [--path <file>] [--property <spec>]...`

## Configuration knobs introduced
None.

## Documentation goals
- Document this command as YAML-only at this stage.

## Implementation goals
- Implement CLI wiring only.

## Automated testing goals
- CLI-layer tests for argument parsing and deterministic errors.

## Acceptance criteria
- CLI delegates to core; schema logic is not duplicated in CLI.
