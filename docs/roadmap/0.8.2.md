# Roadmap 0.5.2 — HTTP: `POST /v1/x402/requirements` (Generate 402 Body)

## Goal
Add a hosted HTTP endpoint that generates the 402 “payment required” response body deterministically from x402 config.

## Progress
planned

## Non-goals
- Any x402 feature logic in core.
- CLI command support (separate patch if desired).

## User-visible outcomes
- `POST /v1/x402/requirements` returns the 402 JSON body deterministically.

## Related previous versions
- `0.5.1` (x402 402 generation logic exists as an HTTP feature)

## New or changed interfaces (exact names + boundaries)
None.

## Module responsibilities (what belongs in a module vs core)
- **Core**: provides unit/state primitives.
- **Hosted HTTP**: owns x402 request handling and 402 generation.

## Data model changes
None.

## API/CLI changes
- `POST /v1/x402/requirements`

## Configuration knobs introduced
None.

## Documentation goals
- Document this endpoint in `docs/spec/hosted-facilitator-http.md` and keep x402 wire rules in `docs/spec/x402.md`.

## Implementation goals
- Implement HTTP handler + OpenAPI wiring only.

## Automated testing goals
- Add HTTP-layer tests for deterministic output and request validation.

## Acceptance criteria
- HTTP does not embed persistence logic; it delegates to core primitives for reads/writes.
