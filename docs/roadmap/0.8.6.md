# Roadmap 0.5.6 — HTTP: `POST /v1/x402/ingest` (Ingest Proof + Record Transaction)

## Goal
Add a hosted HTTP endpoint that ingests x402 proof headers and records a normalized transaction deterministically.

## Progress
planned

## Non-goals
- Any x402 feature logic in core.
- CLI command support (separate patch if desired).

## User-visible outcomes
- `POST /v1/x402/ingest` records a transaction deterministically.

## Related previous versions
- `0.5.3` (x402 header parsing exists as an HTTP feature)
- `0.5.4` (x402 normalization exists as an HTTP feature)
- `0.5.5` (x402 uniqueness exists as an HTTP feature)

## New or changed interfaces (exact names + boundaries)
None.

## Module responsibilities (what belongs in a module vs core)
- **Core**: provides unit/state primitives.
- **Hosted HTTP**: owns ingest pipeline operations and HTTP request/response validation.

## Data model changes
None (delegates to core).

## API/CLI changes
- `POST /v1/x402/ingest`

## Configuration knobs introduced
None (proof storage policy remains feature-owned and config-driven).

## Documentation goals
- Document this endpoint in `docs/spec/hosted-facilitator-http.md` and keep x402 wire rules in `docs/spec/x402.md`.

## Implementation goals
- Implement HTTP handler + OpenAPI wiring only.

## Automated testing goals
- Add HTTP-layer tests for deterministic error messages and request validation.

## Acceptance criteria
- HTTP does not embed persistence logic; it delegates to core primitives for writes.
