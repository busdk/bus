# Roadmap 0.0.4 — Codecs, Canonical Model, and Deterministic Encoding

## Goal
Make multi-format storage (YAML/TOML/JSON) pluggable via codecs, with deterministic encoding and a canonical in-memory representation.

## Progress
planned

## Non-goals
- Preserve comments/whitespace round-trip.
- Define schema validation in full.

## User-visible outcomes
- Workspaces may store manifest/schemas/units in YAML/TOML/JSON.
- Bus behavior is deterministic when multiple candidates exist.

## Related previous versions
- `0.0.3` (manifest exists; workspace roots established)

## New or changed interfaces (exact names + boundaries)
Core-owned codec extension points:
- `Codec`
- `FormatRegistry`

Canonical rules live in:
- `docs/spec/codecs-and-formatting.md`

## Module responsibilities (what belongs in a module vs core)
- **Core**: owns codecs/registry and deterministic encoding rules; the rest of the system does not branch on format.
- **Modules**: read/write through core stores; may add new codecs by implementing and registering `Codec`.

## Data model changes
None (this step defines encoding/format rules, not new domain documents).

## API/CLI changes
Create/init-style commands MUST support:
- `--format yaml|toml|json`

Rule:
- If `--path` includes a recognized extension, extension wins consistently.

## Configuration knobs introduced
- CLI: `--format` for commands that create files.

## Documentation goals
- Document supported formats, codec registry rules, and deterministic encoding rules (single source: `docs/spec/codecs-and-formatting.md`).

## Implementation goals
- Implement codec registry and canonical decode/encode pipeline for manifest, schemas, and unit records.
- Implement deterministic encoding per codec and `.ids` formatting rules.

## Automated testing goals
- Add “parse-equivalence” tests (same logical content across YAML/TOML/JSON).
- Add deterministic encoding tests (byte-for-byte stable output per codec).

## Acceptance criteria
- Decoding the same logical content across formats yields the same canonical representation.
- Encoding is byte-for-byte deterministic per codec.
- `.ids` output obeys the formatting rules.
