# Roadmap 0.0.47 — HTTP: `GET /v1/settlement/proposal` (Future)

## Goal
Add a hosted HTTP endpoint that returns a deterministic settlement proposal by delegating to export/settlement extension points.

## Progress
planned

## Non-goals
- Full settlement automation (payments, invoicing, etc.).
- Selecting a specific external accounting integration.

## User-visible outcomes
- `GET /v1/settlement/proposal` returns a deterministic proposal for a time range and scope.

## Related previous versions
- `0.0.40` (export/settlement extension point interfaces exist)
- `0.0.44` (HTTP server skeleton exists)

## Canonical definitions
- `docs/spec/hosted-facilitator-http.md`
- `docs/spec/exports-and-settlement.md`

## Implementation goals
- Define query params + response schema in OpenAPI.
- Implement handler that calls the settlement proposer/exporter interfaces.

## Acceptance criteria
- Endpoint is tenant-scoped and deterministic.
- Output is stable and versioned under `/v1/...`.
