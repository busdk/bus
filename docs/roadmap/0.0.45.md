# Roadmap 0.0.45 — HTTP: `POST /v1/transactions` (Record Transaction)

## Goal
Add a hosted HTTP endpoint to record a transaction deterministically by delegating to core primitives (schema/unit operations + state backend).

## Progress
planned

## Non-goals
- Defining a new transaction domain model in core.
- Replacing the schema-driven “transaction as unit” pattern.

## User-visible outcomes
- `POST /v1/transactions` records a transaction deterministically.

## Related previous versions
- `0.0.21` (ledger/transactions pattern exists)
- `0.0.44` (HTTP server skeleton exists)

## Canonical definitions
- `docs/spec/hosted-facilitator-http.md`
- `docs/spec/ledger-transactions.md`

## Implementation goals
- Define request/response schema in OpenAPI.
- Implement handler that validates input and calls core unit create/write primitives.

## Acceptance criteria
- Endpoint is tenant-scoped and deterministic.
- HTTP layer performs only transport validation/mapping; persistence is via core primitives.
