# Roadmap 0.0.42 — Core: Operational Automation (Tasks / Playbooks) Interfaces

## Goal
Introduce core-owned interfaces and document-model expectations for **tasks** and **playbooks** that can be attached to units/services and invoked via transports.

## Progress
planned

## Non-goals
- Implementing a full runner/executor.
- Adding a hosted facilitator endpoint surface (separate patch).

## User-visible outcomes
- None (core-only).

## Related previous versions
- `0.0.1` (interfaces + built-in selection)
- `0.0.13` (unit operations)
- `0.0.40` (export/settlement extension points)

## New or changed interfaces (exact names + boundaries)
Operational automation concepts (canonical):
- `docs/spec/operational-automation.md`

## Implementation goals
- Define core-owned interfaces for:
  - listing attached tasks/playbooks
  - invoking a task/playbook (transport calls into core)
- Ensure definitions allow multiple built-in implementations (selected deterministically).

## Acceptance criteria
- Spec defines tasks/playbooks with clear boundaries and attachment targets.
- Interfaces are designed so transports can invoke without embedding feature logic in core.
