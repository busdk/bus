# Roadmap 0.4.2 — CLI: `bus micropayments report`

## Goal
Add the `bus micropayments report` command as a thin CLI wrapper over the core micropayments report operation.

## Progress
planned

## Non-goals
- Any new micropayments logic in core.

## User-visible outcomes
- `bus micropayments report ...` prints deterministic aggregates.

## Related previous versions
- `0.4.1` (core micropayments report operation exists)

## New or changed interfaces (exact names + boundaries)
None.

## Module responsibilities (what belongs in a module vs core)
- **Core**: owns report operation.
- **CLI**: parses args and prints results.

## Data model changes
None.

## API/CLI changes
- `bus micropayments report [--from <ts>] [--to <ts>] [--group-by <dim>]...`

## Configuration knobs introduced
None.

## Documentation goals
- Document this command as a CLI wrapper over core.
  - Link/point to the report aggregation examples in `docs/spec/micropayments.md`.

## Implementation goals
- Implement CLI wiring only.

## Automated testing goals
- Add CLI-layer tests for deterministic output and flag parsing.

## Acceptance criteria
- CLI does not implement reporting logic; it delegates to core.
