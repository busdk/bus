# Roadmap 0.0.43 — Core: Secret Fields (Schema Annotation + Redaction Rules)

## Goal
Define how schemas can mark fields as **secret** and how transports must handle them safely.

## Progress
planned

## Non-goals
- Shipping a specific secret manager integration.
- Perfect redaction in every possible output; defaults must be safe.

## User-visible outcomes
- None (core-only).

## Related previous versions
- `0.0.4` (workspace/manifest exist)
- `0.0.13` (unit create)
- `0.0.15` (unit list/show)
- `0.0.42` (tasks/playbooks reference secrets)

## Canonical definitions
- `docs/spec/secrets.md`

## Implementation goals
- Define schema annotation for “secret” fields.
- Define default redaction behavior for list/show and other outputs.
- Define the secret resolution extension point interface (built-in implementations selected deterministically).

## Acceptance criteria
- Spec defines how secret fields are represented in schema + units (no plaintext by default).
- Default behavior is safe: secrets are not printed unless explicitly requested.
