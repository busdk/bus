# Roadmap 0.3.5 — Core: JSON Codec

## Goal
Add JSON support as an additional codec (deterministic encoding) for users who want JSON config files.

## Progress
planned

## Non-goals
- Make JSON required for the minimal path (YAML remains the default).
- TOML support (separate patch).

## User-visible outcomes
- None (core-only).

## Related previous versions
- `0.1.1` (YAML codec exists first)

## New or changed interfaces (exact names + boundaries)
JSON codec rules (canonical):
- `docs/spec/codec-json.md`

## Module responsibilities (what belongs in a module vs core)
- **Core**: includes JSON codec implementation.

## Data model changes
None.

## API/CLI changes
None (core-only patch).

## Configuration knobs introduced
None.

## Documentation goals
- Ensure JSON codec requirements are documented in `docs/spec/codec-json.md`.

## Implementation goals
- Implement JSON deterministic encoding (pretty-print, stable key ordering, newline).

## Automated testing goals
- Unit tests for deterministic JSON output and decode/encode round-trip.

## Acceptance criteria
- JSON encoding is deterministic across runs.
