# Roadmap 0.0.3 — Workspace Discovery, Manifest, and Tenant Isolation

## Goal
Define how Bus finds a workspace manifest, resolves paths, and enforces tenant isolation without creating tool-defined top-level directories outside `.bus/`.

## Progress
planned

## Non-goals
- Define schema/unit document formats in detail.
- Define format codecs.

## User-visible outcomes
- `bus init` creates a manifest in the current directory.
- Commands error deterministically when the manifest is missing or ambiguous.
- All reads/writes are tenant-scoped.

## Related previous versions
- `0.0.2` (CLI request context carries tenant id; routing exists to invoke capabilities)

## New or changed interfaces (exact names + boundaries)
Core-owned contracts introduced/clarified:
- Workspace discovery + manifest candidate rules
- Tenant selection + isolation rules

Canonical definitions live in:
- `docs/spec/workspace-manifest.md`
- `docs/spec/tenancy.md`

## Module responsibilities (what belongs in a module vs core)
- **Core**: workspace discovery, path resolution, tenant resolution rules, tenant-scoped storage roots.
- **Modules**: implement commands like `init` using core stores; do not write arbitrary files.

## Data model changes
Manifest document (`kind: bus.manifest`) is required. See:
- `docs/spec/workspace-manifest.md`

Tenant-scoped storage root is established:
- `.bus/tenants/<tenantId>/`

## API/CLI changes
- CLI supports `--tenant <TENANT_ID>` (flag introduced in `0.0.2`, defaults defined here).

## Configuration knobs introduced
- Manifest: `tenants.default` (optional; if absent, default tenant is `"default"`).

## Documentation goals
- Document manifest candidate rules, path resolution, and tenant defaulting/isolation (single sources: `docs/spec/workspace-manifest.md`, `docs/spec/tenancy.md`).

## Implementation goals
- Implement manifest discovery in the current directory with deterministic ambiguity errors.
- Implement tenant defaulting rules and tenant-scoped `.bus/tenants/<tenantId>/` roots.
- Implement `bus init` to create a minimal manifest and `.bus/` structure.

## Automated testing goals
- Add tests for manifest discovery (none/one/multiple candidates).
- Add tests for tenant defaulting order and storage root isolation.

## Acceptance criteria
- Manifest discovery matches candidate rules exactly.
- Tenant defaulting matches the resolution order.
- No command requires creating top-level directories outside `.bus/`.
