# Roadmap 0.2.9 — CLI Command: `bus <schema> show`

## Goal
Add the `bus [schema] <schema> [unit] <id> [show]` CLI command as a thin wrapper over core unit show.

## Progress
planned

## Non-goals
- Any new show logic in core.

## User-visible outcomes
- `bus <schema> unit <id>` same as `bus schema <schema> unit <id> show`
- `bus <schema> <id>` same as `bus schema <schema> unit <id> show`, unless if `<id>` conflicts with any built-in keyword
  - Conflicts with UUID and integer numbers should be less likely since no built-in Bus argument will have a `-` or start with a number
  - Conflicts with user-defined primary string keys are possible. However, it's strongly suggested to use only keys that would never 
    conflict, like domain or host names. For example `bus domains example.com` would never overlap with a built-in argument since the primary key has a dot in it.
- `bus schema <schema> unit <id> show` prints a unit record deterministically.
